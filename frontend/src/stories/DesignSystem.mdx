# Mini-UPS Design System

Welcome to the Mini-UPS design system documentation. This system is built using the **shadcn/ui** approach, combining **Radix UI** primitives with **Tailwind CSS** for styling and **Class Variance Authority (CVA)** for component variants.

## 🎨 Design Philosophy

Our design system follows these core principles:

### 1. **Accessibility First**
- Built on Radix UI primitives ensuring WCAG 2.1 compliance
- Keyboard navigation support
- Screen reader compatibility
- Proper ARIA attributes

### 2. **Consistency**
- Unified visual language across the application
- Standardized spacing, typography, and color usage
- Predictable component behavior

### 3. **Flexibility**
- Composable components that can be easily extended
- Multiple variants for different use cases
- Customizable through CSS classes and props

### 4. **Performance**
- Lightweight components with minimal bundle impact
- Tree-shakeable exports
- Optimized for production builds

## 🎯 Usage Guidelines

### Do's ✅
- Use semantic HTML elements

- Implement consistent hover and focus states

### Don'ts ❌
- Don't override component internals
- Don't use hardcoded colors outside the design system
- Don't create one-off components without design system consideration
- Don't ignore accessibility requirements

## 🔧 Customization

### Extending Components
Components can be extended using the `className` prop:

```tsx
<Button className="w-full" variant="outline" size="lg">
  Extended Button
</Button>
```

### Creating New Variants
Use CVA to add new variants:

```tsx
const buttonVariants = cva(
  "base-styles",
  {
    variants: {
      variant: {
        // Add new variant
        special: "bg-special text-special-foreground"
      }
    }
  }
)
```
